# リグレッションテスト (qsub.py / sbatch.py)

`qsub.py` / `sbatch.py` の `--dry-run` 出力（生成ジョブスクリプト+バリデーションエラー+exit code）を
`golden/` のスナップショットと比較するゴールデンテストです。デプロイ前に実行し、
ジョブスクリプト生成が意図せず変わっていないことを確認します。

GPU・クラスタ・ネットワーク不要（`--dry-run` はジョブスクリプトを表示するだけでファイルも書きません）。

```bash
cd evaluation/installers/intg-eval-2508/tests

# 比較 (デプロイ前後・変更後に実行)
bash run_regression.sh

# 意図した変更の場合はスナップショットを更新し、diffをレビューしてコミット
bash run_regression.sh --update
git diff golden/
```

- テストケースは `cases.txt` に1行1ケース (`<名前> <スクリプト> <引数...>`) で定義します。
  新しいオプションや分岐を追加したら、対応するケースと (`--update` で) スナップショットを追加してください。
- 環境変数はランナー内でダミー値に固定しているため出力は決定的です (実シークレットは不要かつ使用禁止)。
- エラーケースは Traceback の行番号情報を除去し、例外メッセージと exit code のみを比較します。
- なお、これはリポジトリ側の生成ロジックの回帰検知のみです。実験ディレクトリへ**デプロイ済み**の
  環境がリポジトリと一致しているかの検証は別途必要です (`<experiment_dir>/environment/` 配下と
  インストーラがコピーするファイルの diff 確認)。
